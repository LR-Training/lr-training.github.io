Cucumber testing stack

There are a number of technologies that make up the testing stack. Below there is a brief summary of what is used

Cucumber

Cucumber is the framework used for Behavioural Driven Development (BDD). It allows the automation of scenarios created in the 'Gherkin' language by writing the code behind the steps to enable the actions. These are the step definitions and the code is written in ruby utilising the capybara library for web development. The automated tests against a web interface are executed using a form of web driver. Cucumber home page

Gherkin scenarios

Gherkin is the 'Given, When, Then' language used to write the behaviours that are being tested. The language should be readable by both developers and end users. This comes with the install of cucumber and they are known as features. An example of gherkin
